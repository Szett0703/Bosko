================================================================================
NOTAS IMPORTANTES PARA EL BACKEND
================================================================================

1. CONTRASEÑAS: Las contraseñas proporcionadas son sin hashear. El backend
   debe aplicar el algoritmo de hash correspondiente (BCrypt recomendado).

2. ROLES: Implementar correctamente:
   - Admin: Acceso total al panel de administración
   - Customer: Acceso a funciones de cliente (compras, perfil, órdenes)

3. ESTADOS DE ÓRDENES: Usar exactamente estos valores en minúsculas:
   - pending, processing, shipped, delivered, cancelled

4. MÉTODOS DE PAGO: Usar snake_case:
   - credit_card, debit_card, paypal

5. FECHAS: Formato ISO 8601 (ejemplo: 2025-11-19T14:30:00Z)

6. IMÁGENES: Los productos deben tener URLs de imágenes placeholder o reales
   en el campo ImageUrl.

7. VALIDACIONES:
   - Emails únicos
   - Stock no negativo
   - Precios mayores a 0
   - Al menos una dirección predeterminada por usuario
